01 — Problem & Approach

Why does shortest path get hard with multiple objectives?

With a single cost metric, shortest path is solved in O((V+E) log V). Add a second objective — say time alongside distance — and the Pareto front can grow exponentially. We sidestep exact enumeration by introducing an adaptive ε-vector that dynamically coarsens the approximation grid per objective, bounding the Pareto front to polynomial size while guaranteeing (1+ε)-approximation for each cost dimension.

STEP 01
Problem Formulation
Given a directed graph G = (V, E) with k ≥ 2 non-negative cost functions c₁, …, cₖ on edges, find all Pareto-optimal s–t paths under the lexicographic dominance order.
NP-Hard for k ≥ 2
STEP 02
ε-Approximation Grid
Scale cost values into logarithmic buckets of width (1+εᵢ) per objective i. Two paths land in the same bucket-vector iff their costs agree to within multiplicative factor (1+εᵢ) on each dimension.
Key Insight
STEP 03
Adaptive Scaling
εᵢ is not fixed globally — it adapts per objective based on the observed cost range in the current subgraph. Tight ranges use fine grids; wide ranges use coarser grids, balancing accuracy and runtime.
Novel Contribution
STEP 04
Label-Setting Expansion
Modified Dijkstra propagates bucket-label vectors. Dominance pruning discards labels that are dominated in the rounded space, keeping the label set polynomial in |V|, 1/εᵢ, and log(Cₘₐₓ).

04 — Complexity Analysis

Formal guarantees & bounds

The Adaptive ε-Vector FPTAS achieves polynomial time and solution-set size under mild assumptions. The table below compares it with prior work.

Time: O( |V| · |E| · (1/εₘᵢₙ)^(k-1) · log^k(C_max / c_min) )
Solutions: O( (1/εₘᵢₙ)^(k-1) · log^(k-1)(C_max) )
Approximation: ∀ exact solution x*, ∃ approximate ŷ s.t. cᵢ(ŷ) ≤ (1+εᵢ) · cᵢ(x*) for all i
Algorithm Time Complexity Guarantee Adaptive ε?
Exact (k=2) O(|V| · |E| · C_max) Exact —
Parametric Dijkstra O(|E| · |V| log |V|) Exact (bi-obj) No
Uniform ε-FPTAS O(|V||E|(1/ε)^(k-1)) (1+ε) uniform No
Adaptive ε-Vector FPTAS ★ O(|V||E|(1/εₘᵢₙ)^(k-1) log^k C) (1+εᵢ) per-obj Yes

The adaptive scheme is particularly powerful when objectives have heterogeneous cost ranges — a common scenario in transportation networks where time spans seconds to hours while monetary cost spans cents to hundreds. By tuning εᵢ independently, the algorithm allocates approximation budget where it matters.
